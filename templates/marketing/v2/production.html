§ 01 — four pre-production failures

Four failures every long-form production inherits.

failure 01

The eight-week breakdown.

Script breakdowns are manual and slow. Every department re-tags the script for their own needs, duplicating work and missing each other's notes.

failure 02

The VFX bid.

Vendors quote from shot lists made by manually scanning each scene. Miss a VFX need, and unexpected costs appear; over-tag, and the show seems more expensive than it is. Getting this wrong is costly.

failure 03

The siloed deliverable.

Each department gets a flat CSV; there’s no shared identity layer. Every revision, teams redo their own breakdowns, losing cross-scene and cross-episode connections each time.

failure 04

The draft-comparison gap.

New drafts show page changes, but not shifts in continuity—like missing props or changed character states. These unseen gaps appear during production.

§ 01a — what the streamers are already building

The major streamers are investing engineering here.

Netflix presented its AI Breakdown Assistant in 2024 — a scene-level VFX classifier and an algorithmic complexity-tier model that takes shot media attributes, script embeddings, and scene understanding as inputs to budget estimation. Their own headline benefits, on the slide:

  • Production time savings via reduced manual work
  • Standardised taxonomy for metrics engineering to measure production complexity
  • More accurate budget estimations, leading to better production planning
  • Connected data to inform creative departments downstream

That's the brief. Fabula does the deeper version of it: entity continuity preserved across episodes and seasons (not just within a script), per-event character state attached to every participation, narrative connections typed and queryable, every claim traceable to the line of script that justified it. If your studio isn't building this internally, the alternative is buying a pre-built version that's run end-to-end on 800+ episodes already.

§ 02a — synthesis you'd actually circulate

Coverage that updates with the draft, sourced to the line.

Most AI coverage tools summarise your script in one pass—useful for triage, but not for ongoing work. Any revision means starting over.

Fabula builds from the graph instead: episode synopses, season arcs, and character summaries all draw from structured events and cite script lines underneath. Update a draft and only the changed sections regenerate.

In development meetings, you get defensible prose: if someone asks what changes about Cromwell between seasons?, the answer traces back to the page.

§ 03 — for ai builders

Building AI over stories? Here’s your retrieval target.

The Fabula graph has 25 node types, 44 relationships, and links every data point to its original script line. Events connect through 10 narrative relations—CAUSAL, FORESHADOWING, THEMATIC_PARALLEL, CALLBACK, ESCALATION, and others. Each connection records its type, strength, and a short explanation—making story logic queryable in ways transcripts and embeddings alone can’t.

Characters' beliefs, goals, emotions, and states are tracked per event, not just in aggregate — so LLMs receive focused, sourced context for any question, not raw transcripts. That distinction matters more than it used to. Long-context LLMs degrade across long inputs in ways that are hard to detect and harder to audit; structured retrieval over a typed graph sidesteps the problem by never asking a single call to hold the whole show in mind. For GraphRAG: the Parquet loads directly into Neo4j, typed edges included. For data science: just open the Parquet in your notebook and start exploring.

56 datasets live on the brandburner org on Hugging Face — the full Doctor Who megagraph (703 episodes), the TNG megagraph, the West Wing megagraph, and per-series exports for Wolf Hall, Happy Valley, Indiana Jones, Dracula, Knives Out, and I, Claudius. Query the hosted Parquet directly via the Datasets Server (DuckDB / Polars / SQL), or download once and load into whatever stack you're running. There's no Fabula API to depend on; the graph is the artefact. 

// GraphRAG retrieval · The West Wing · typed connections + provenance

MATCH (anchor:Event {scene_anchor: 'S2E14 · Act III'})
      -[c:THEMATIC_PARALLEL|CALLBACK|EMOTIONAL_ECHO]-(echo:Event)
      -[:OCCURS_IN]->(ep:Episode)
RETURN ep.code              AS episode,
       echo.scene_anchor    AS scene,
       type(c)               AS connection,
       c.strength            AS strength,
       c.description         AS why,
       echo.script_lines     AS provenance
ORDER BY c.strength DESC;

// One typed traversal returns the scenes an LLM should read to
// answer "where else has the show set this up, and how did it pay
// off?" — not via vector similarity, via the show's own narrative
// architecture. Every row cites the script lines verbatim.
